# Feature Creator Agent — MVP (Streamlit)

Минимальное приложение: загрузка одного CSV, превью и базовая статистика, затем сжатое описание датасета отправляется в LLM. В ответе — интерпретация данных и идеи новых признаков для классического ML и нейросетей. **Обучение моделей не выполняется.**

## Требования

- **Python 3.10** или новее (нужны синтаксис и стандартная библиотека уровня 3.10+).
- Ключ API для OpenAI или совместимого эндпоинта (через переменные окружения в `.env`).

## Установка и запуск

Выполните в каталоге репозитория по порядку.

**1. Виртуальное окружение**

```bash
cd feature_creator_agent
python -m venv .venv
```

Если команда `python` недоступна, используйте `python3 -m venv .venv`.

**2. Активация**

- macOS / Linux:

```bash
source .venv/bin/activate
```

- Windows (cmd):

```bat
.venv\Scripts\activate.bat
```

- Windows (PowerShell):

```powershell
.venv\Scripts\Activate.ps1
```

**3. Зависимости**

```bash
pip install -r requirements.txt
```

**4. Переменные окружения**

Скопируйте шаблон и отредактируйте `.env` (как минимум ключ API):

```bash
cp .env.example .env
```

Укажите `OPENAI_API_KEY` или `LLM_API_KEY`, при необходимости `OPENAI_MODEL`, `OPENAI_BASE_URL`, `LLM_TIMEOUT_SEC`, `LLM_MAX_OUTPUT_TOKENS`, `CSV_MAX_MB`, `CSV_PROFILE_MAX_ROWS`, `CSV_PROFILE_SAMPLE_MODE`.

Для модели **`openai/gpt-oss-120b`** см. раздел ниже — её нельзя вызвать через `api.openai.com`.

**5. Запуск приложения**

```bash
streamlit run app.py
```

В терминале появится локальный URL (часто `http://localhost:8501`) — откройте его в браузере.

### Модель `openai/gpt-oss-120b`

Это **open-weight** модель; на **официальном** OpenAI API (`https://api.openai.com/v1`) она **не** предоставляется. Нужен провайдер с **OpenAI-совместимым** эндпоинтом `…/v1/chat/completions`.

Примеры настройки в `.env` (ключ берите у выбранного сервиса; это уже не `sk-…` от OpenAI):

- **Together AI** (`OPENAI_BASE_URL=https://api.together.xyz/v1`, `OPENAI_MODEL=openai/gpt-oss-120b`).
- **OpenRouter** (`OPENAI_BASE_URL=https://openrouter.ai/api/v1`, та же модель). По желанию задайте `OPENROUTER_SITE_URL` и `OPENROUTER_APP_TITLE` — некоторые провайдеры используют их как метаданные запроса.

Бесплатность («free») зависит только от **выбранного провайдера** (пробные кредиты, акции и т.д.), не от этого репозитория.

### Прокси и ошибка 407

Если при вызове LLM появляется **407 Proxy Authentication Required**, задайте в `.env` **`LLM_PROXY`** с учётными данными, например:

`LLM_PROXY=http://login:password@proxy.company.local:8080`

Символы `@`, `:`, `#` и др. в пароле нужно **URL-кодировать**. Если в macOS/Windows в переменных окружения уже висит `HTTPS_PROXY` **без** логина и он конфликтует с корпоративным прокси, добавьте **`LLM_IGNORE_SYSTEM_PROXY=1`** и оставьте только `LLM_PROXY` с полным URL.

## Поведение

- **Лимит размера CSV** задаётся в `.env` параметром `CSV_MAX_MB` (по умолчанию 15 МБ).
- **Профиль для LLM** считается не более чем по `CSV_PROFILE_MAX_ROWS` строкам (по умолчанию 50 000): режим `CSV_PROFILE_SAMPLE_MODE=first` (первые строки) или `random` (воспроизводимая выборка, `CSV_PROFILE_SAMPLE_SEED`). В JSON есть `source_total_rows` и блок `profile_sampling`.
- **Бинарные ячейки** в столбцах приводятся к коротким строкам (или плейсхолдеру `<binary …>`); имена таких столбцов перечислены в `binary_columns_normalized`.
- В UI показаны предупреждения о передаче данных на **внешний API** и о **секретах** в файле (токены, пароли и т.п.).
- Чтение CSV: перебор кодировок (UTF-8, BOM, CP1251, Latin-1), разделитель **авто** (первая непустая строка) или **вручную** в боковой панели (`;`, таб, `|` и т.д.).
- В LLM уходит только агрегированное описание: имена столбцов, типы, примеры значений, число строк, пропуски, уникальность по столбцам (полные строки таблицы не отправляются).
- Вызов API: **httpx** к OpenAI-совместимому `POST .../chat/completions`; приоритет `response_format=json_object`, при отказе провайдера — повтор без режима и разбор JSON из текста. Таймаут и `max_tokens` задаются через `LLM_TIMEOUT_SEC` и `LLM_MAX_OUTPUT_TOKENS`. Ошибки пишутся в лог **без** ключей.

## Поток работы (end-to-end)

1. Пользователь загружает `.csv` и при необходимости выбирает **разделитель** в боковой панели (или оставляет **Авто** — эвристика по первой непустой строке).
2. Приложение проверяет размер файла, пустоту и «текстовость», затем читает таблицу (`read_csv_bytes` с перебором кодировок).
3. Строится **профиль** (лимит строк, нормализация бинарных ячеек) и отображается на вкладках «Превью» и «Профиль».
4. После согласия с отправкой данных на внешний API и нажатия **«Проанализировать»** профиль уходит в **LLM**, ответ показывается на вкладке **«Вывод ИИ»**.

## Ручные сценарии тестирования

1. **Пустой файл и лимит размера**  
   Создайте пустой `empty.csv` (0 байт) или файл из одних пробелов — ожидайте понятное сообщение об ошибке. Затем временно уменьшите `CSV_MAX_MB` в `.env` и загрузите заведомо большой CSV — должно сработать ограничение по размеру без чтения в профиль.

2. **Разделитель и «не-CSV»**  
   Подготовьте файл `test.csv` с полями через **точку с запятой** (`a;b\n1;2`) — с режимом **Авто** таблица должна разобраться на два столбца. Переименуйте небольшой бинарный файл в `.csv` и загрузите его — ожидайте ошибку про двоичное содержимое или неудачный разбор; при текстовом CSV с нестандартным разделителем выберите разделитель **вручную** в сайдбаре.

3. **Ошибка API**  
   Временно укажите в `.env` неверный `OPENAI_API_KEY` или недоступный `OPENAI_BASE_URL`, загрузите валидный CSV, отметьте согласие и нажмите **«Проанализировать»** — ожидайте сообщение об ошибке API и подсказку проверить ключ, URL и сеть.

## Структура репозитория

- `app.py` — точка входа Streamlit
- `src/csv_profile.py` — чтение CSV из байтов, таблица профиля столбцов, JSON-сводка для LLM
- `src/llm_client.py` — HTTP-вызов чат-API
- `src/prompts.py` — системный и пользовательский текст промптов
- `requirements.txt` — зависимости
- `.env.example` — шаблон переменных окружения
