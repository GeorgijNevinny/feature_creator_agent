--------------------------------------------------------------------------------
СЛАЙД 1. Название
--------------------------------------------------------------------------------

Feature Creator
Автоматический feature engineering с помощью цепочки AI-агентов

Веб-приложение на Streamlit: загрузили CSV — получили признаки
и отчёт с проверкой на моделях.


--------------------------------------------------------------------------------
СЛАЙД 2. Проблема
--------------------------------------------------------------------------------

Подготовка признаков для ML — один из самых трудоёмких этапов пайплайна:

  • аналитик вручную изучает схему CSV и роли столбцов;
  • data scientist придумывает формулы и проверяет гипотезы;
  • инженер валидирует признаки на нескольких моделях.

Это занимает дни, требует доменной экспертизы и легко упускает
сильные признаки или тратит время на слабые.


--------------------------------------------------------------------------------
СЛАЙД 3. Решение
--------------------------------------------------------------------------------

Feature Creator — веб-приложение, в которое загружается один CSV-файл.
Три специализированных AI-агента работают последовательно:

  1) анализируют данные и назначают роли столбцам;
  2) генерируют осмысленные признаки с формулами и обоснованием;
  3) проверяют каждый признак на табличных и нейросетевых моделях.

На выходе — готовый отчёт: сводка по датасету, список признаков,
метрики кросс-валидации, превью обогащённой таблицы, экспорт JSON.


--------------------------------------------------------------------------------
СЛАЙД 4. Что делает продукт
--------------------------------------------------------------------------------

  • Понимает предметную область данных (финансы, телеком, retail и др.)
  • Автоматически определяет роли столбцов: id, timestamp, target, features
  • Предлагает признаки под задачу — с формулой и объяснением «зачем»
  • Считает признаки на вашем файле и показывает превью enriched-таблицы
  • Отбирает только те признаки, которые дают прирост метрики на CV
  • Работает с произвольным CSV — без ручной настройки под каждый кейс

Один прогон заменяет типичный цикл «разбор данных → ideation → ablation».


--------------------------------------------------------------------------------
СЛАЙД 5. Архитектура: три агента
--------------------------------------------------------------------------------

  ┌─────────────────────┐
  │ Data Analyst Agent  │  Профиль CSV: типы, пропуски, статистика,
  │                     │  домен, гипотеза ML-задачи, роли столбцов
  └──────────┬──────────┘
             ▼
  ┌─────────────────────┐
  │ Feature Engineering │  Генерация признаков: имя, формула, rationale,
  │ Agent               │  применимость (tabular / neural networks)
  └──────────┬──────────┘
             ▼
  ┌─────────────────────┐
  │ ML Validation Agent │  Обучение: LightGBM, XGBoost, MLP (PyTorch)
  │                     │  5-fold CV, ROC-AUC, статус passed / marginal
  └─────────────────────┘

Агенты обмениваются контекстом: аналитик передаёт схему инженеру,
инженер — список кандидатов валидатору. На экране — карточки статуса,
пошаговый лог и прогресс выполнения.


--------------------------------------------------------------------------------
СЛАЙД 6. Интерфейс
--------------------------------------------------------------------------------

Главный экран:

  • заголовок Feature Creator и описание возможностей;
  • загрузка CSV (до заданного лимита размера);
  • превью первых 10 строк исходных данных;
  • кнопка «Запустить агентов».

Боковая панель:

  • разделитель полей CSV (авто / запятая / ; / таб / |);
  • «Быстрый режим» — ускоренный прогон с тем же качеством отбора.

После завершения пайплайна:

  • сводка по датасету (домен, что измеряется, тип ML-задачи);
  • таблица ролей столбцов;
  • сгенерированные признаки;
  • валидация на моделях (baseline → with_feature → delta);
  • превью обогащённых данных с новыми колонками;
  • скачивание отчёта feature_creator_report.json.


--------------------------------------------------------------------------------
СЛАЙД 7. Живая демонстрация
--------------------------------------------------------------------------------

Шаг 1. Запуск:

  cd feature_creator_demo
  python -m venv .venv && source .venv/bin/activate
  pip install -r requirements.txt
  streamlit run app.py

Шаг 2. Загрузить CSV — например eda/tesla.csv
       (~1510 торговых дней, OHLCV Tesla, 2019–2024).

Шаг 3. Нажать «Запустить агентов».

       Агенты последовательно анализируют файл, строят признаки
       и прогоняют кросс-валидацию — обычно 2–3 минуты.

Шаг 4. Разобрать отчёт: сводка, 5 признаков, таблица валидации,
       скачать JSON при необходимости.


--------------------------------------------------------------------------------
СЛАЙД 8. Пример: датасет Tesla
--------------------------------------------------------------------------------

Что загружаем: дневные котировки акций Tesla (OHLCV).

Столбцы: date, open, high, low, close, adj_close, volume.

Что определяет Data Analyst Agent:

  • домен — дневные биржевые ряды;
  • задача — классификация направления close на следующий день (up/down);
  • date — временная ось для лагов и скользящих окон;
  • OHLCV — базовые признаки, close — опора для таргета и доходностей.


--------------------------------------------------------------------------------
СЛАЙД 9. Пример: признаки, которые построил Feature Engineering Agent
--------------------------------------------------------------------------------

На данных Tesla агенты предложили и посчитали пять признаков:

1. gap_from_prev_close_pct
   (open - close_prev) / close_prev
   Overnight gap — реакция рынка на события вне сессии.

2. vwap_proxy_dev
   Отклонение close от typical price (high+low+close)/3.
   Внутридневное давление покупателей и продавцов.

3. volatility_20d
   Стандартное отклонение дневных доходностей за 20 дней.
   Режим волатильности — спокойный vs турбулентный рынок.

4. intraday_range_pct
   (high - low) / close
   Нормированный дневной диапазон, «ширина» свечи.

5. volume_zscore_20
   Z-score объёма по окну 20 дней.
   Аномальная ликвидность и новостные всплески.

Каждый признак сопровождается формулой, rationale и пометкой,
подходит ли он для tree-based моделей и нейросетей.


--------------------------------------------------------------------------------
СЛАЙД 10. Пример: результаты ML Validation Agent (Tesla)
--------------------------------------------------------------------------------

Валидатор обучает модели с признаком и без него, сравнивает метрики
на 5-fold time-series cross-validation. Метрика — ROC-AUC.

Лучшие результаты (LightGBM):

  gap_from_prev_close_pct   baseline 0.504 → with_feature 0.524  (+0.020)
  vwap_proxy_dev            baseline 0.504 → with_feature 0.523  (+0.020)
  intraday_range_pct        baseline 0.504 → with_feature 0.521  (+0.018)

Также проверены XGBoost и MLP (PyTorch). Статус passed означает
устойчивый прирост к baseline; marginal — слабый или нестабильный эффект
на данной архитектуре (такой признак можно отложить).

Итог прогона: отобран набор признаков, которые реально улучшают
прогноз направления цены, а не только «красиво выглядят в отчёте».


--------------------------------------------------------------------------------
СЛАЙД 11. Как это работает под капотом
--------------------------------------------------------------------------------

  1. Загрузка CSV → парсинг, профилирование столбцов (типы, пропуски,
     распределения, корреляции).

  2. Data Analyst Agent получает сжатое описание датасета и возвращает
     интерпретацию домена, гипотезу ML-задачи и роли столбцов.

  3. Feature Engineering Agent на основе профиля и ролей генерирует
     кандидаты признаков, затем они вычисляются на полном датафрейме.

  4. ML Validation Agent для каждого признака:
       • строит таргет (например, next-day direction);
       • обучает LightGBM, XGBoost, MLP с baseline-набором и с новым признаком;
       • считает ROC-AUC по фолдам и присваивает статус отбора.

  5. Результаты собираются в единый JSON-отчёт и отображаются в UI.

Всё выполняется в одном сеансе — от сырого файла до проверенных признаков.


--------------------------------------------------------------------------------
СЛАЙД 12. Технологический стек
--------------------------------------------------------------------------------

  • Python 3.10+
  • Streamlit — веб-интерфейс
  • pandas — загрузка CSV, feature computation, превью данных
  • scikit-learn — препроцессинг и кросс-валидация
  • LightGBM, XGBoost — градиентный бустинг на табличных признаках
  • PyTorch — MLP для проверки признаков в нейросетевом сценарии
  • LLM-агенты — анализ схемы и генерация идей признаков

Артефакты на выходе: обогащённый датафрейм (превью), JSON-отчёт,
готовый к передаче в downstream ML-пайплайн или feature store.


--------------------------------------------------------------------------------
СЛАЙД 13. Кому полезно и что дальше
--------------------------------------------------------------------------------

Целевая аудитория:

  • data scientist и ML-инженер — ускорение feature ideation;
  • аналитик данных — быстрый разбор нового CSV;
  • продуктовые команды — прототип ML на реальных таблицах без недель ручной работы.

Планы развития:

  • пакетная обработка нескольких файлов;
  • экспорт в Parquet / feature store;
  • интеграция с MLOps (Airflow, MLflow);
  • расширение набора моделей и метрик под регрессию и мультикласс.


--------------------------------------------------------------------------------
СЛАЙД 14. Итог
--------------------------------------------------------------------------------

Feature Creator сокращает путь от «сырой CSV» к «проверенным признакам
с метриками на моделях» — в одном интерфейсе, за один прогон.

Три агента закрывают весь цикл: понять данные → придумать признаки →
отфильтровать слабые на реальном обучении.

Загрузите свой файл — и получите отчёт, готовый к использованию
в моделировании.
